# Changelog 2026-03-07

本日修改围绕 **internvla_runtime（gRPC 服务）** 与 **真实世界部署（Unitree Go2）**：补全同事留下的 worker、打通 C++ 构建、新增 Go2 gRPC 客户端与部署说明。

---

## 1. internvla_runtime / Python worker（worker_server.py）

- **完整 args**：用 argparse 提供 `device`、`model_path`（默认 `checkpoints/InternVLA-N1-DualVLN`）、`resize_w`、`resize_h`、`num_history`、`plan_step_gap`、`port`，并设置 `camera_intrinsic`。
- **InitEpisode**：实现按 `session_id` 存储 `instruction`；客户端通过本 RPC 传入导航指令。
- **Step**：传入 `intrinsic`、从 `_session_instruction` 取 instruction（不再写死），并做 warmup（与 http server 一致）。
- **响应**：支持 `pixel_x`/`pixel_y`，轨迹用 `np.asarray` 保证类型正确。

---

## 2. Proto 与生成脚本

- **gen_proto.sh**：从 `proto/internvla.proto` 生成 Python 的 `internvla_pb2.py`、`internvla_pb2_grpc.py`（`internvla_runtime/python/`）。
- **gen_proto_cpp.sh**：生成 C++ 的 `internvla.pb.cc/.h`、`internvla.grpc.pb.cc/.h`（`internvla_runtime/cpp/gen/`）。

---

## 3. C++ 构建（CMakeLists + 文档）

- **CMakeLists.txt**：加入 `cpp/gen` 下生成的 `.cc` 与 `target_include_directories`；当系统无 `gRPCConfig.cmake` 时用 **pkg-config** 查找 `grpc++` 并创建 `gRPC::grpc++` 目标，解决 Ubuntu 上 CMake 找不到 gRPC 的问题。
- **readme.txt**：补充 C++ 依赖、生成与编译步骤、常见问题（g++ 用法、rtl8812au-dkms）。

---

## 4. 客户端与示例

- **client_example.py**：在 `internvla_runtime/python/` 下新增最小 gRPC 示例（InitEpisode + Step，假数据），用于验证 worker。
- **grpc_internvla_client.py**：在 `scripts/realworld/` 下新增 **Unitree Go2 真相机客户端**，与 `http_internvla_client.py` 同流程（ROS2 话题、MPC/PID 控制），改为 gRPC 调用 worker；支持 `--instruction`、`--grpc` 命令行参数。
- **http_internvla_client.py**：文件头注明 Unitree Go2 + HTTP；逻辑未改。
