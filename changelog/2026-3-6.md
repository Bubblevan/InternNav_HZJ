# Changelog 2026-03-06

本日修改围绕 **realworld HTTP 服务**（`python3 scripts/realworld/http_internvla_server.py`）的启动与运行问题，涉及 4 个文件。

---

## 1. `scripts/realworld/http_internvla_server.py`

- **新增参数** `--plan_step_gap`（默认 `4`），供 `InternVLAN1AsyncAgent` 使用，避免 `AttributeError: 'Namespace' object has no attribute 'plan_step_gap'`。
- **Warmup 调用** `agent.step(...)` 时补充传入 `intrinsic=args.camera_intrinsic`，满足 `step()` 必选参数，避免 `TypeError: ... missing 1 required positional argument: 'intrinsic'`。
- **Warmup 输入数据类型**：RGB 改为 `np.uint8`、depth 改为 `np.float32`，与 PIL / 实际请求一致，避免 `TypeError: Cannot handle this data type: (1, 1, 3), <f8>`。
- **默认模型路径** 由 `checkpoints/InternVLA-N1` 改为 `checkpoints/InternVLA-N1-DualVLN`，默认加载带 system1 的 DualVLN（system2+system1）。

---

## 2. `internnav/agent/internvla_n1_agent_realworld.py`

- 在 `__init__` 中设置 `self.save_dir` 后增加 `os.makedirs(self.save_dir, exist_ok=True)`，确保调试图片保存目录存在，避免 `FileNotFoundError: ... debug_raw_xxx.jpg`。

---

## 3. `internnav/model/basemodel/internvla_n1/internvla_n1_arch.py`

- 在 `InternVLAN1MetaModel.__init__` 末尾增加 **兜底逻辑**：若 `config` 无 `system1`（例如从部分 checkpoint 加载时未包含该字段），仍会创建 `latent_queries` 参数（`n_query` 优先取 `config.n_query`，否则默认 `4`；`hidden_size` 取 `config.hidden_size`），避免 forward 时报 `AttributeError: 'InternVLAN1Model' object has no attribute 'latent_queries'`。使用带 `system1` 的 config（如 DualVLN）时行为与原先一致。

---

## 4. `internnav/model/utils/vln_utils.py`

- 在 `traj_to_actions` 中，对 `dp_actions` 在转 numpy 前先执行 `float().detach()`，再做 unnormalize 与 `cpu().numpy()`，避免推理时出现 `RuntimeError: Can't call numpy() on Tensor that requires grad. Use tensor.detach().numpy() instead.`。

---

## 小结

| 文件 | 改动要点 |
|------|----------|
| `http_internvla_server.py` | 增加 `plan_step_gap`、warmup 传 `intrinsic`、warmup 用 uint8/float32、默认模型改为 DualVLN |
| `internvla_n1_agent_realworld.py` | 启动时创建 `save_dir` 目录 |
| `internvla_n1_arch.py` | 无 `system1` 时兜底创建 `latent_queries` |
| `vln_utils.py` | `traj_to_actions` 中 `dp_actions` 先 detach 再转 numpy |
