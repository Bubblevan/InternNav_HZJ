# 学长反馈执行指南

## 学长说了什么（简要）

1. **需要看**：第一视角视频 + 模型输出结果  
2. **正常行为**：一开始会原地旋转，之后大部分时间应输出 pixel 或什么都不输出（memory）  
3. **狗走得慢**：不是 server 的问题，要先确认「给定一条轨迹，狗能否跟上」  
4. **解决方向**：调 MPC 的 target v（目标速度）

---

## 一、给学长看「第一视角视频」和「模型输出」

### 1.1 第一视角视频

**方式 A：实时看**

- Server 运行时，在浏览器打开：`http://<server_ip>:5801/fpv_live`  
- 例如：`http://192.168.1.105:5801/fpv_live`  
- 会显示当前帧画面（含红点标注的 pixel_goal）

**方式 B：录制成视频**

- Server 每帧会保存到：`stream/<timestamp>/frame_000001.jpg`, `frame_000002.jpg` ...  
- 路径示例：`/home/ubuntu1/backup/InternNav/stream/2026-03-08-143113/`  
- 用 ffmpeg 合成视频：

```bash
cd /home/ubuntu1/backup/InternNav/stream/2026-03-08-143113
ffmpeg -framerate 10 -i frame_%06d.jpg -c:v libx264 -pix_fmt yuv420p fpv_output.mp4
```

把 `fpv_output.mp4` 发给学长。

### 1.2 模型输出

- **Server 日志**：`scripts/realworld/logs/internvla_server_*.log`  
- 每行包含：`action_type`（trajectory / discrete_action）、`action_value`、`pixel_goal`、`traj_len`  
- 可直接把对应时间段的 server 日志发给学长，或配合 `analyze_frame_logs.py` 的统计结果

## 三、调 MPC 的 target v（Client 端，解决「走得慢」）

### 3.1 参数在哪

- **MPC 控制器**：`controllers.py` 中 `Mpc_controller.__init__`  
- **创建 MPC 的位置**：`http_internvla_client.py` 约第 248 行  

当前默认：

- `desired_v=0.3`：目标线速度 (m/s)  
- `v_max=0.4`：最大线速度 (m/s)  

### 3.2 已修改

已在 `http_internvla_client.py` 中改为 `desired_v=1.2, v_max=1.5`，并在 `controllers.py` 中将 MPC 成本 R[0,0] 从 0.05 降为 0.01，减少对速度的惩罚。

---

## 四、固定轨迹测试：验证「给定轨迹，狗能否跟上」

已内置测试模式，**无需启动 Server**，在 Unitree 上执行：

```bash
cd ~/backup/InternNav/scripts/realworld
INTERNVLA_FIXED_TRAJ_TEST=1 python3 http_internvla_client.py
```

- 会注入一条 **L 形轨迹**：直走 4m → 左转 90° → 往左走 2m
- 观察能否稳定跟上、转弯是否顺畅、速度是否合适
- 用 `Ctrl+C` 结束
