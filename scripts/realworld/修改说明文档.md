# InternVLA 实机 Client 与 MPC 修改说明文档

## 一、出发点与背景

### 1.1 问题来源

在 Unitree Go2 上部署 InternVLA DualVLN 系统时，遇到以下问题：

1. **狗走得慢**：即使将 `desired_v` 调高至 2.0，实际运动速度仍接近 0.8 m/s，与预期不符。
2. **需验证 MPC 跟踪能力**：学长建议先确认「给定一条固定轨迹，狗能否稳定跟上」，再排查模型/指令问题。

### 1.2 修改目标

- 提高 MPC 实际输出速度，使 `desired_v` 能有效体现。
- 增加固定轨迹测试模式，便于单独验证 MPC 跟踪能力。

## 三、修改原因

### 3.1 Server URL

- **原因**：Client 在 Unitree、Server 在 PC 时，`127.0.0.1` 指向 Unitree 本机，无法连到 Server。
- **处理**：默认使用 Server 所在机器 IP（如 `192.168.1.105`），同机部署可用 `INTERNVLA_SERVER_URL=http://127.0.0.1:5801` 覆盖。

### 3.2 MPC 速度偏慢

- **原因**：MPC 成本函数为 `J = Σ(tracking_error) + Σ(R·u²)`，其中 `R[0,0]=0.05` 对 v² 惩罚较大，优化倾向于使用较小速度。
- **处理**：将 `R[0,0]` 降为 `0.01`，使 MPC 更愿意提高线速度。

### 3.3 `find_reference_traj` 弧长计算

- **原因**：原逻辑用 `cum_dist[i] - cum_dist[nearest_idx]` 计算从最近点到参考点的弧长。当 `nearest_idx > 0` 时，`cum_dist[nearest_idx]` 表示到 `pt(nearest_idx+1)` 的弧长，导致参考点选取偏前，后半段轨迹被漏选。
- **处理**：改为 `cum_dist_from_nearest = cum_dist[nearest_idx-1]`（`nearest_idx > 0`），正确计算从 `pt(nearest_idx)` 出发的弧长。

### 3.4 固定轨迹测试模式

- **原因**：需在无 Server 的情况下验证 MPC 能否跟上给定轨迹，排除模型/指令影响。
- **处理**：通过环境变量 `INTERNVLA_FIXED_TRAJ_TEST=1` 启用，注入 L 形轨迹，不发起 HTTP 请求。

## 四、如何启动 http_internvla_client.py

### 6.1 正常模式（连 Server）

**前置条件**：

- Server 已在 PC 上运行：`python3 http_internvla_server.py`
- Unitree 已配置 ROS 2 环境（如 Foxy）
- 相机、odom 等话题正常

**启动命令**（在 Unitree 上）：

```bash
cd ~/backup/InternNav/scripts/realworld
python3 http_internvla_client.py
```

若 Server 不在 `192.168.1.105`，可设置环境变量：

```bash
INTERNVLA_SERVER_URL=http://<Server_IP>:5801 python3 http_internvla_client.py
```

### 6.2 固定轨迹测试模式（不连 Server）

**用途**：单独验证 MPC 跟踪能力，无需启动 Server。

**启动命令**（在 Unitree 上）：

```bash
cd ~/backup/InternNav/scripts/realworld
INTERNVLA_FIXED_TRAJ_TEST=1 python3 http_internvla_client.py
```

**行为**：

- 注入 L 形轨迹：直走 4m → 左转 90° → 往左走 2m
- 狗按该轨迹运动，用 `Ctrl+C` 结束

## 参数速查

| 参数 | 位置 | 当前值 | 说明 |
|------|------|--------|------|
| `desired_v` | Client 创建 MPC 时 | 1.2 | 目标线速度 (m/s) |
| `v_max` | Client 创建 MPC 时 | 1.5 | 最大线速度 (m/s) |
| `R[0,0]` | controllers.py | 0.01 | 线速度惩罚权重，越小越愿加速 |
| `INTERNVLA_SERVER_URL` | 环境变量 | 默认 192.168.1.105:5801 | Server 地址 |
| `INTERNVLA_FIXED_TRAJ_TEST` | 环境变量 | 0/1 | 1 时启用固定轨迹测试 |
